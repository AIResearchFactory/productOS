7. Visual journey boards

Concrete UX flows showing screen transitions, guidance, approval, and recovery moments.

Journey A

First product setup

From empty workspace to first useful output without learning Skills or Workflows.

1

Create product

Name the product. Home starts with one CTA: Add context.

2

Add context

Upload, paste, or import source material. Readiness becomes visible.

3

Create output

ProductOS recommends PRD, one-pager, roadmap, or research summary.

4

Review draft

Parker shows attached context and asks before saving.

5

Automate

Home suggests recurring automation once value is proven.


Journey B

Mature product daily review

1

Open Home

See readiness, recent changes, stale outputs, and automations.

2

Follow rec

Next best action points at the stale or highest-leverage item.

3

Use Parker

Ask for product status or let Parker draft the update.

4

Approve

Review diff/source trace before saving generated outputs.

5

Audit

Source trace and run history explain what happened and why.
