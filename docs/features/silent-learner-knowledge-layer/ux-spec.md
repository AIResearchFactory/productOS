# UX Spec: Silent Learner Knowledge Layer

## Overview

The Silent Learner Knowledge Layer operates **silently by design**. The UX footprint is minimal — most operations happen in the background without interrupting the user. Visibility is provided through existing UI surfaces (project settings, artifacts panel).

**Related documents:**
- [PRD](/docs/features/silent-learner-knowledge-layer/prd.md)
- [Existing Silent Learner UX](/docs/features/silent-learner/ux.md)

---

## Primary User Flow: Import-to-Learn

```
User drops files → Files appear in project → [Background: enrichment runs]
                                              → [Background: sidecars created]
                                              → [Background: index.md updated]
User starts AI chat → Better responses (context from enriched data)
                   → [Background: scores update]
                   → [Background: knowledge pages created when thresholds met]
User opens project settings → Sees file scores (compounding visible here)
                           → Can trigger "Optimize Memory" for manual enrichment
```

**Key UX principle:** The user should never need to take any action for the knowledge layer to work. All enrichment, knowledge building, and maintenance is autonomous.

## Screen States

### Artifacts Panel (Existing)
- **New:** Knowledge pages (`knowledge/*.md`), lessons (`lessons/*.md`), and syntheses (`syntheses/*.md`) appear as regular artifacts
- **Label:** These should be visually distinguishable as "Auto-generated" (subtle badge or icon)
- **Interaction:** Users can read, but edits are overwritten on next observation cycle
- **Empty state:** No knowledge artifacts until Silent Learner observes enough interactions

### Project Settings — Silent Learner Section (Existing)
- **No new controls needed** — existing score panel shows file scores
- **Enhancement:** Scores should reflect the new `silentLearner.compositeScore` from enriched sidecars
- **Enhancement:** Add "Knowledge Health" summary (count of lint findings by severity) below the score panel

### Knowledge Health Panel (New, in Project Settings)
- **Trigger:** Visible when Silent Learner is enabled and lint findings exist
- **Content:** Grouped findings by type (orphans, stale, duplicates)
- **Actions:** Each finding can be dismissed or acted upon (e.g., "Create knowledge page" link)
- **State: Empty** — "No issues found. Your knowledge base is healthy. ✓"
- **State: Findings** — List with severity icons (info/warning)

---

## Accessibility Requirements

- Knowledge health findings must be keyboard-navigable
- Auto-generated artifact badges must have `aria-label="Auto-generated by Silent Learner"`
- Score values in project settings must be screen-reader accessible
