# Backend Plan: PWA Migration

## Summary
The Axum backend must natively serve the built React files instead of depending purely on hot-reload/external static servers for the final delivery.

## API Spec
- `/api/*`: Existing application endpoints for Chat, Projects, Config, System.
- `/*`: Fallback routed to statically serve the `dist/` directory generated by Vite.

## Implementation Details
1. Add `tower-http` with the `fs` feature to `src-tauri/Cargo.toml` if it isn't deployed already.
2. In `src-tauri/src/server/main.rs` (or routing module), use `tower_http::services::ServeDir::new("../dist")`.
3. Ensure fallback `ServeFile` routes unmatched URLs to `index.html` to support React client-side routing natively.

## Validation Rules
The local-first policy mandates no internal CORS warnings since frontend and backend operate on a single domain + port (e.g., `127.0.0.1:51423`).

## Dependencies
Requires `tower-http` `fs` flag enabled.
