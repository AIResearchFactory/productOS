# TEAM.md — productOS Operating Team

## Purpose
This is the working team structure for growing productOS across product, marketing, sales, architecture, and quality.

## Agents / Roles

### 1. Product Strategist
Owns roadmap, prioritization, user value, packaging, and competitive framing.

### 2. Growth & Marketing Lead
Owns demos, positioning, messaging, onboarding narrative, and launch content.

### 3. Sales Engineer
Owns solution storytelling, objections, ROI framing, and customer-facing technical explanation.

### 4. Product Architect
Owns system design, execution decomposition, technical review quality, and release architecture.

### 5. Builder & QA Lead
Owns implementation tracking, testing coverage, regression control, and release readiness.

## Working mode
- All new initiatives begin as proposed tasks in BACKLOG.md
- Human approval is required before execution of new major workstreams
- Daily iteration should update DAILY.md with what changed, blockers, and next actions
- New skills/tools/files that expand capability require explicit approval before installation
