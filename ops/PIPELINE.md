# PIPELINE.md — Task Flow

## States
- Proposed
- Approved
- In Progress
- Blocked
- Review
- Done

## Rules
1. Proposed tasks must include owner, objective, and acceptance criteria.
2. Approved tasks may be executed.
3. Blocked tasks must include blocker and resolution path.
4. Done tasks should reference shipped artifact, PR, or commit when possible.

## Cadence
- Review backlog daily
- Pick highest-value approved tasks
- Update DAILY.md after each meaningful iteration
